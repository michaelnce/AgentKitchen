Chapter One

The Problem

Agent orchestration is abstract. Documentation alone doesn't build intuition.

👨‍🍳

What's an Agent?

A coordinator that breaks tasks into parts, delegates work, and assembles results.

🧑‍🔬

What's a Sub-Agent?

A specialized worker that runs independently and returns structured data.

📋

What's a Skill?

A reusable, deterministic transformation. No reasoning — just formatting or conversion.

AgentKitchen makes these patterns concrete and runnable.

Chapter Five

Sub-Agent vs Skill

This is the core teaching moment of the recipe.

Sub-Agent

  • Uses LLM reasoning
  • Generates new content
  • Returns structured JSON
  • Can use tools (web search)
  • Output varies per run
vs

Skill

  • Pure transformation
  • No reasoning needed
  • Deterministic output
  • Like a template engine
  • Same input = same output

FormatRecipeCard is a skill — it just plates the dish. No cooking required.

Chapter Six

REVIEW The TopChef Pattern

The final tasting — a quality gate that cross-references multiple sources.

What TopChef Receives

  • 1. The formatted recipe card
  • 2. Web research findings (sources, consensus, tips)

What TopChef Does

  • ✓ Fixes ingredients that conflict with sources
  • ✓ Incorporates best techniques
  • ✓ Notes controversies with reasoning
  • ✓ Adds Chef's Tips + Sources

Chapter Ten

The Techniques

1. Orchestrator Pattern

A head chef that delegates, collects, and assembles. The ChefAgent doesn't cook — it coordinates.

2. Parallel Execution

Independent stations work concurrently. 4 agents launch at once, cutting total time.

3. Contract Communication

Stations return JSON, not free text. Agents communicate through structured tickets.

4. Review Gate

A final tasting that merges multiple sources before the dish leaves the kitchen.

5. Tool-Using Agent

RecipeResearcher uses WebSearch and WebFetch to ground output in real-world data.

6. Skill vs Agent

Skills plate. Agents cook. Knowing when to use which is the mark of a great kitchen.
